## 🚀 发布步骤详解

### 第一步：开发者注册（一次性费用 $5 USD）

1. **访问开发者控制台**
   - 打开：https://chrome.google.com/webstore/devconsole
   - 使用Google账号登录

2. **支付开发者费用**
   - 点击"注册为开发者"
   - 支付一次性费用 $5 USD
   - 完成身份验证

### 第二步：创建新扩展项目

1. **点击"新增项目"**
   - 在开发者控制台点击"New Item"
   - 选择"上传扩展程序"

2. **上传扩展包**
   - 上传 `gordon-translator-v1.2.2.zip`
   - 等待系统验证

### 第三步：填写扩展信息

#### 基本信息
- **名称**: Gordon Translator
- **简短描述**: 现代化的 Google 翻译客户端，支持智能主题切换和选择即翻译
- **详细描述**: 
```
Gordon Translator 是一个现代化的 Google 翻译客户端，专为 Chrome 浏览器设计。

🌓 智能主题切换
- 支持浅色、深色和自动主题
- 根据系统时间自动切换主题

⚡ 快速翻译
- 集成 Google Translate API
- 支持多种语言互译

🎯 选择即翻译
- 在任意网页选择文本即可翻译
- 自动打开侧边栏显示翻译结果

🎨 现代化界面
- 简洁美观的设计
- 响应式布局

隐私保护：仅调用 Google Translate API，不收集用户数据。
```

#### 分类和语言
- **类别**: 效率工具 (Productivity)
- **语言**: 中文（简体）、英文

### 第四步：上传资源文件

#### 商店图标
- 使用 `icons/icon128.png`（128x128像素）
- 确保图标清晰，符合Chrome设计规范

#### 截图（至少1张，建议3-5张）
使用 `Images/` 目录下的图片：
- `HEADER_1280_800.png` - 主界面截图
- `HEADER_1400_560.png` - 功能展示截图
- `HEADER_440_280.png` - 小尺寸截图

**截图要求**：
- 尺寸：1280x800 像素
- 格式：PNG 或 JPEG
- 内容：清晰展示扩展的主要功能

### 第五步：隐私和权限设置

#### 隐私政策
在"隐私实践"部分声明：
- ✅ 不收集个人信息
- ✅ 仅使用 Google Translate API 进行翻译
- ✅ 所有设置仅保存在本地浏览器
- ✅ 不跟踪用户行为

#### 权限说明
- `storage`: 保存用户设置
- `sidePanel`: 显示翻译面板
- `activeTab`: 获取当前页面信息
- `scripting`: 注入选择翻译功能
- `host_permissions`: 访问 Google Translate API

### 第六步：发布设置

1. **可见性设置**
   - 选择"公开"（Public）
   - 扩展将对所有用户可见

2. **地区设置**
   - 选择"全球"或指定地区
   - 建议选择全球以扩大用户群

3. **定价**
   - 选择"免费"

### 第七步：提交审核

1. **最终检查**
   - 确认所有信息填写完整
   - 检查截图和图标质量
   - 验证扩展功能正常

2. **提交审核**
   - 点击"提交审核"
   - 等待Google审核（通常1-3个工作日）

## 📝 审核注意事项

### 常见审核失败原因
1. **功能问题**：扩展无法正常工作
2. **权限过度**：请求不必要的权限
3. **描述不符**：实际功能与描述不符
4. **截图质量**：截图模糊或无法展示功能
5. **隐私问题**：未正确声明数据使用

### 提高审核通过率的建议
1. **充分测试**：在多个网站测试扩展功能
2. **权限最小化**：只请求必要的权限
3. **描述准确**：确保描述与实际功能一致
4. **截图清晰**：提供高质量的截图
5. **隐私透明**：明确说明数据使用情况

## 🔄 版本更新流程

### 更新步骤
1. **修改代码**：实现新功能或修复bug
2. **更新版本号**：修改 `manifest.json` 中的 `version`
3. **重新打包**：创建新的zip文件
4. **上传更新**：在开发者控制台上传新版本
5. **提交审核**：等待审核通过

### 版本号规范
- 主版本号：重大功能更新（1.0.0 → 2.0.0）
- 次版本号：新功能添加（1.0.0 → 1.1.0）
- 修订号：bug修复（1.0.0 → 1.0.1）
